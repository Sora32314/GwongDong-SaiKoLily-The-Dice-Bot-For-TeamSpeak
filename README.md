## 简介

**GwongDong-SaiKoLily** 是运行在 TeamSpeak 3 客户端上的掷骰插件（BOT），专门为 **TRPG** 玩家设计。  
输入简单的骰子表达式，即可在频道内实时获得掷骰结果，同时支持会话管理、团队记录与历史导出。

> ✨ 无论你是在跑《克苏鲁的呼唤》还是《龙与地下城》，在TeamSpeak平台的小广东都能帮你快速处理投掷！

## 快速开始

### 环境要求

| 依赖 | 版本 / 说明 |
|------|-------------|
| TeamSpeak 3 Client | 支持 Plugin API v26 的客户端 |
| C++ 编译器 | 支持 C++20（MSVC 2022、GCC 11+、Clang 16+） |
| spdlog | 1.15.x（将通过 CMake 自动获取） |
| CMake | ≥ 3.25 |
| Ninja | 推荐（跨平台构建加速） |

> ⚠️ 如果你的 TeamSpeak 3 客户端比较旧，请先升级到支持 API v26 的版本。

### 构建项目

本工程使用 **CMake Presets** 管理构建，需要 CMake ≥ 3.25 并安装 Ninja。

#### 🔨 Debug 构建

```bash
# 1. 配置
cmake --preset "core debug"

# 2. 编译
cmake --build --preset "Debug Build"
# 或者直接指定目录
cmake --build Build/Debug
```

#### 🚀 Release 构建

```bash
cmake --preset "core release"
cmake --build Build/Release
```

编译成功后，插件文件将生成在 `Build/Release`（或 `Debug`）目录下，复制到 TeamSpeak 3 的插件目录即可加载。

## 使用指南

### 掷骰命令

所有掷骰命令均支持前缀 `[. ! >]`，方便不同习惯的玩家。

| 命令 | 别名 | 说明 | 示例 |
|------|------|------|------|
| `!r <表达式>` | `!roll` / `!rd` | 标准掷骰（支持多骰加减） | `!r 2d6+1d4-3` |
| `.r <表达式>` | `.roll` / `.rd` | 同上，点前缀 | `.roll 1d20+5` |
| `>r <表达式>` | `>roll` / `>rd` | 同上，尖括号前缀 | `>rd 3d10` |

**支持的表达式语法：**

- `XdY`：投掷 X 个 Y 面骰（例如 `2d6`）
- 加减运算：`2d6+1d4-3`
- 单个常量：`3`（直接返回 3）
- 多个骰子组合：`1d20+5-1d4`

### 会话管理命令

所有会话命令均以前缀 `.ss` 开头。

#### 📂 会话生命周期

| 命令 | 说明 | 示例 |
|------|------|------|
| `.ss create <标题> [描述]` | 创建一个新会话，可附带队伍公告 | `.ss create "黑水溪" "团本：黑水溪，车卡需求：……"` |
| `.ss end [方式] [ID]` | 结束一个会话 | `.ss end index 3` 或 `.ss end title 黑水溪` |
| `.ss select <方式> <参数>` | 选择一个已有会话作为当前活动会话 | `.ss select id 1` 或 `.ss select title 新手跑团` |
| `.ss unselect` | 取消当前选择的会话 | `.ss unselect` |

#### 👥 成员管理

| 命令 | 说明 | 示例 |
|------|------|------|
| `.ss join <用户名...>` | 将一个或多个用户加入当前会话 | `.ss join "AMX-10" "电子魏特曼"` |
| `.ss quit <用户名...>` | 将一个或多个用户移出当前会话 | `.ss quit "BVVD的木琴" "滚木"` |

#### 📊 信息查询与导出

| 命令 | 说明 | 示例 |
|------|------|------|
| `.ss info` | 查看当前会话的详细信息（标题、描述、成员、投掷次数等） | `.ss info` |
| `.ss list` | 列出服务器上所有活跃的会话 | `.ss list` |
| `.ss history [n]` | 查看当前会话最近 n 次投掷记录（默认 10） | `.ss history 20` |
| `.ss save [路径] [文件名] [扩展名]` | 将当前会话的完整投掷历史保存为文件 | `.ss save ./logs 跑团记录 txt` 或 `.ss save ./logs LOGS dummy` |

### 插件的工作方式非常简单，六步即可上手：

1. **加载插件**  
   将编译好的插件文件放入 TeamSpeak 3 客户端插件目录，启动客户端并启用插件。  

2. **进入频道**  
   当你用来 _HOST_ 插件的客户端正常进入某个服务器的频道后，插件会自动获取频道内成员列表，以便在投掷时准确识别是谁在丢骰子。  

3. **开始投掷**  
   在频道聊天框输入 `!r 2d6+1d4-3` 之类的指令，插件会根据当前环境决定如何处理：
   - 如果你已经通过 `.ss select` 选择了某个会话 _（必须要首先加入会话才能选择）_  → 使用该会话的记录引擎（**高质量随机数**）
   - 如果你加入了某个会话（即使没有主动选择） → **同样使用你加入的会话**
   - 如果当前频道绑定了某个会话 → **使用该频道会话**
   - **以上都不满足** → 插件会自动创建一个临时的默认骰子系统，并采用**低质量随机数引擎**快速生成结果且记录 _**不会被保存**_

4. **（极度推荐）加入或创建会话**  
   为了保证投掷结果的统计学真实性，强烈建议使用 `.ss create` 创建一个跑团会话，或用 `.ss join` 加入一个已有的会话并且使用 `.ss Select` 选择一个会话（因为你可以加入多个会话，为了让插件分辨你到底加入的是那个会话，请主动选择，选择只需要一次，可以随时使用 `.ss Select` 或 `.ss Unselect` 更换你所选择的会话）。  
   只要进入会话环境，插件就会自动切换到**高质量随机数生成器（RNG）**，让你的骰点更具可信度。

5. **实时记录与回溯**  
   会话中每一次掷骰都会被自动记录。  
   使用 `.ss history` 可以随时翻看最近的投掷，`.ss save` 更能把完整记录导出为文件，方便赛后复盘。

6. **结束会话**  
   跑团结束后，使用 `.ss end` 关闭会话，释放资源。

> 🧩 **选择优先级：用户主动选择（Select） > 用户所在会话 > 频道绑定的会话。**  
> 这样即使你加入了多个会话，也能明确知道当前骰子记录归属。

## 输出示例

**输入**：
```
!r 2d6+1d4-3
```

**频道内输出**：
```
投掷结果：2d6[3, 5]+1d4[2]-3
点数：7
```

**会话历史记录**（`.ss history`）中会类似显示：
```
[2026-04-27 20:30:15] Sora32314: 2d6+1d4-3 = [3,5]+[2]-3 = 7
```

## 配置与日志

- **日志文件**：默认输出到 `logs/async_logs.txt`，包含详细运行信息和错误堆栈。
- **服务器连接**：插件启动时自动连接 `localhost`。需要修改目标服务器 IP / 端口，请编辑 `Init_Plugin.cpp` 中的相关参数。
- 其他可配置项（如历史记录条数上限、保存格式）可在 `config.h` 中调整。（暂未支持，等待后续开发）
